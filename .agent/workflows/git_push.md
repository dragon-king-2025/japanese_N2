# Git Push Workflow

This workflow stages all changes, generates a commit message based on the current diff, and pushes to the remote repository.

## Steps

1. **Check current status** to understand what has changed:
```powershell
git -C "c:\projects\japanese_N2" status --short
```

2. **Stage all changes**:
// turbo
```powershell
git -C "c:\projects\japanese_N2" add .
```

3. **Analyze the diff** to auto-generate a commit message. Review the staged changes:
```powershell
git -C "c:\projects\japanese_N2" diff --cached --stat
```

4. **Generate and execute the commit**. Based on the diff analysis, create a concise commit message in the format:
   - First line: short summary (max 72 chars), e.g. `feat: unified 7-column vocab tables for Day 01-14`
   - Body (optional): bullet points of major changes
// turbo
```powershell
git -C "c:\projects\japanese_N2" commit -m "<auto-generated message>"
```

5. **Push to remote**:
// turbo
```powershell
git -C "c:\projects\japanese_N2" push
```

## Notes
- The commit message should be generated by examining the staged diff and summarizing the key changes.
- Use conventional commit prefixes: `feat:`, `fix:`, `refactor:`, `docs:`, `style:`, `chore:`.
- Always run from the `c:\projects\japanese_N2` repository root.
